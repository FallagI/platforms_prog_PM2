

Демонстрационный проект для иллюстрации рабочего процесса GitHub Flow.

## Структура проекта

- `main.py` - главный модуль приложения
- `auth.py` - модуль аутентификации пользователей
- `README.md` - документация проекта

## Используемый workflow

### Основные ветки

1. **main** - основная ветка, всегда содержит работоспособный код
2. **feature/** - ветки для разработки новой функциональности
3. **hotfix/** - ветки для срочных исправлений
4. **docs/** - ветки для обновления документации

### Процесс разработки

1. Создать новую ветку от `main`
2. Внести изменения и сделать коммиты
3. Создать Pull Request
4. Пройти код-ревью
5. Слить изменения в `main`
